PRD: Plataforma de Quiniela Mundial 20261. Resumen del ProyectoDesarrollar una aplicación web para gestionar una quiniela de fútbol enfocada en la Primera Fase (Fase de Grupos) del Mundial, permitiendo a los usuarios pronosticar resultados de partidos y eventos especiales del torneo para competir por una tabla de posiciones.2. Stack TecnológicoFrontend: React.js (SaaS moderno, interactividad en tiempo real).Backend: Django / Django REST Framework (Lógica de negocio y procesamiento de puntos).Base de Datos: Supabase (PostgreSQL + Auth + Realtime).Infraestructura: Conexión vía ORM de Django a la DB de Supabase.3. Requerimientos FuncionalesRF1: Gestión de SesionesRegistro e inicio de sesión de usuarios gestionado por Supabase Auth.Perfil de usuario básico vinculado a sus pronósticos.RF2: Pronósticos de Fase de GruposFormulario dinámico en React para cargar los 48 juegos de la primera fase.Capacidad de editar pronósticos antes del cierre.RF3: Predicciones EspecialesSección para seleccionar Campeón, Subcampeón, 3er/4to lugar, Goleador y Asistente.Uso de listas desplegables (dropdowns) para evitar errores de escritura en nombres de jugadores/equipos.RF4: Regla de Cierre "Hard Lock"Bloqueo Total: Al iniciar el primer juego del mundial, se deshabilita automáticamente la opción de crear o modificar cualquier pronóstico (tanto de partidos como especiales).Validación tanto en Frontend (UI bloqueada) como en Backend (403 Forbidden).RF5: Cálculo y RankingPanel de administración (Django Admin) para que el administrador cargue los resultados reales.Script de procesamiento masivo para actualizar puntos_obtenidos en la tabla de Supabase.Leaderboard: Tabla de posiciones global ordenada por puntaje total.5. Roadmap de DesarrolloFase 1: Configuración de Supabase y conexión con Django.Fase 2: Creación de API para listar partidos de la fase de grupos.Fase 3: Desarrollo en React del formulario de carga (Cartelera).Fase 4: Implementación de la lógica de bloqueo por fecha/hora.Fase 5: Sistema de cálculo de puntos y Leaderboard.
